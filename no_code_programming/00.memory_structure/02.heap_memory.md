# Heap memory

stack 의 경우에는 메모리의 더 낮은 주소로 sequentially 자란다. 그렇지만 heap 의 경우에는 그 순서나 위치가 불분명하다. 따라서 heap 메모리를 건들 때는, 순서가 있는 stack 위에 포인터를 만든 후 heap의 객체를 포인팅 하는게 좋다.

Heap 메모리가 필요한 이유
1. Life Cycle

    stack 메모리의 가장 큰 문제 중 하나는 life-cycle 이 있다는 거임. 우리가 프로그램에 장기적으로 필요한 resource를 쭉 가져가고 싶다면 heap 에 할당을 해놓고 stack 위에 있는 pointer 만 신경 쓰면 되니까 편함.
2. Large Size

    스택 메모리의 공간 크기는 heap 에 비해 상대적으로 작음.
3. Dynamic (runtime)

    프로그램 실행 과정에서 결정할 수 있다.
