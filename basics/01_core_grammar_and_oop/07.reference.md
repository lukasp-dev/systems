In Cpp, understanding the immutability of a reference's binding is fundamental. Unlike a pointer, which is a variable that holds a memory address, a reference is an alias for an existing object.

ONce a reference is initialized, it is "bonded" to that object for life. You cannot "re-seat" or reassign it to refer to a different memory location.
if `std::string &nickname = original_name;`, then `nickname` is `original_name`. In kernel-level programming, we often use this to pass heavy structures without the overhead of copying, while ensuring the identity of the object doesn't shift unexpectedly during execution.

```cpp
int a = 10;
int b = 20;

int& ref = a;
ret = b;
```

In the example above, `ref=b` is functionally the same thing as writing `a=b`. You aren't changing what `ref` refer to; you are modifying the value of the underlying object `a`.

### Modern C++ reference model (interview-grade)

### 1) Lvalue reference vs Rvalue reference
- `T&` binds to lvalues (named objects).
- `T&&` binds to rvalues (temporaries, expiring values).

```cpp
std::string s = "hello";
std::string&  lref = s;                 // OK
const std::string& cref = std::string("tmp"); // lifetime extension
std::string&& rref = std::string("tmp");      // OK
```

### 2) Lifetime extension rule
- Binding a temporary to `const T&` extends temporary lifetime to the reference scope.
- Same idea applies to `T&&` local bindings.
- Important: lifetime extension does not happen through raw pointers.

### 3) Forwarding reference (`T&&` in template context)
In `template<class T> void f(T&& x)`, `T&&` is a forwarding reference.
It preserves value category with `std::forward<T>(x)`.

```cpp
template <class T>
void wrapper(T&& x) {
    consume(std::forward<T>(x)); // keeps lvalue/rvalue nature
}
```

### 4) Interview heuristics
1. Read-only large object: `const T&`
2. Sink/ownership transfer: `T&&` or by-value + move
3. Small trivials (`int`, `double`, pointers): pass by value

### Full source (07.reference.cpp)

```cpp
#include <iostream>
#include <vector>

using namespace std;

void work(int &x){
    x++;
}

void work(vector<int>& vec){
    vec[0]++;
}

int main(){
    vector<int> data = {10, 10, 30, 50, 40, 20, 430, 40, 2};

    cout << data[0] << "\n"; //10
    work(data);
    cout << data[0] << "\n"; //11
    
    int a = 10;
    int& b = a; // This is how you create the alias
    int c = 100;

    cout << a << " " << b << "\n"; //10 10
    b = c;
    cout << &a << " " << &b << "\n"; //0x16b71e698 0x16b71e698
    work(a); 
    cout << a << " " << b << "\n"; // 101 101

    return 0;
}
```
