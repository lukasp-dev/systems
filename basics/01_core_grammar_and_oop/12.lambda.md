# Lambda Expression Structures

### C++ Lambda Expression Structure

| Component            | Name               | Description                                                                          |
| :------------------- | :----------------- | :----------------------------------------------------------------------------------- |
| **`[ ]`**            | **Capture Clause** | Defines which variables from the outside scope are available inside the lambda.      |
| **`( )`**            | **Parameters**     | Defines the input arguments, just like a regular function. (Optional)                |
| **`-> return_type`** | **Return Type**    | Specifies the type of the value returned. (Optional; compiler can often infer this). |
| **`{ }`**            | **Function Body**  | Contains the actual logic/code to be executed.                                       |

---

### Syntax Overview

```cpp
[ capture_clause ] ( parameters ) -> return_type {
    // function_body
};
```

### Quick Examples in Code

```cpp
// 1. Basic Lambda
auto greet = []() { std::cout << "Hello!";}

// 2. Lambda with Parameters and Return Type
auto add = [](int a, int b) ->int {
    return a + b;
};

// 3. Generic Lambda (Using 'auto' in parameters)
auto print_data = [](const auto& data) {
    std::cout << data << std::endl;
}

```

### Capture Clauses

### C++ Lambda Capture Clause Summary

| Capture Syntax | Name                 | Description                                                                              |
| :------------- | :------------------- | :--------------------------------------------------------------------------------------- |
| **`[ ]`**      | **Empty**            | No local variables are captured. Only global or static variables are accessible.         |
| **`[x]`**      | **By Value**         | Captures a **copy** of `x`. The original value outside remains unchanged.                |
| **`[&x]`**     | **By Reference**     | Captures a **reference** to `x`. Changes inside the lambda affect the original variable. |
| **`[=]`**      | **All by Value**     | Captures all local variables used in the body **by value** (copy).                       |
| **`[&]`**      | **All by Reference** | Captures all local variables used in the body **by reference**.                          |
| **`[x, &y]`**  | **Mixed**            | Captures `x` by value and `y` by reference.                                              |
| **`[this]`**   | **This Pointer**     | Captures the **current class instance**. Allows access to member variables/functions.    |

---

### Key Concepts

- **Local vs. Global**: Lambdas inhabit a separate scope. You must explicitly "capture" local variables from `main()` or other functions to use them.
- **Lifetime**: Be careful with **Reference Capture (`[&]`)**; if the lambda outlives the captured variable (e.g., the function ends), it will point to invalid memory (Dangling Reference).
- **Immutable by Default**: Variables captured by value `[x]` are `const` by default inside the lambda. To modify the copy, use the `mutable` keyword.

### Full source (12.lambda.cpp)

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    int factor = 2;
    int total = 0;
    std::vector<int> numbers = {1, 2, 3, 4, 5};

    // Example 1: Capture by Value [factor]
    // 'factor' is copied. If 'factor' changes outside later, the lambda keeps the old value.
    auto multiply_by_val = [factor](int n) {
        return n * factor; 
    };

    // Example 2: Capture by Reference [&total]
    // We can modify 'total' directly inside the lambda.
    std::for_each(numbers.begin(), numbers.end(), [&](int n) {
        total += n; // Modifies the actual 'total' variable outside
    });

    // Example 3: Mixed Capture [factor, &total]
    // 'factor' is read-only copy, 'total' is modifiable reference.
    auto mixed_capture = [factor, &total]() {
        total *= factor;
    };

    std::cout << "Sum: " << total << "\n"; // 15
    mixed_capture();
    std::cout << "Sum after doubling: " << total << "\n"; // 30

    return 0;
}
```
