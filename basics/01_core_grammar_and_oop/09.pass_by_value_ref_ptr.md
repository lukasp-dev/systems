By default, Cpp passes by value by default.
But there is one exception, arrays decay to pointer.

#### pass by value
Copying the entire box.
- Mechanism: creates a complete, independent copy of the argument int he function's stack frame.
- Memory View: The function gets a new memory address with the same bit-pattern as the original
- Low level detail: Triggers **Copy Constructors** for objects.
    - For large structures (`vectors` for example), this involves heap allocation and O(n) data movement.
    - Use Case: Small primitive (`int`, `char`, `bool`) or when you need to modify data without affecting the caller.
    - Pros/Cons: Safe, but potentially very expensive in terms of latency and memory.
#### pass by reference
This is like **giving the box the new nickname**.
- Mechanism: THe function parameter becomes a symbolic alias for the original variable.
- Memory View: No new object is created. The compiler treats the parameter name as the original memory location.
- Lowe-level Detail: **No Reassignment:** The alias is permanently bonded to the original memory address at compile time.
    - No Dereferencing(*) Syntax
    - Optimization: Highly efficient. For large objects, it passes an 8 byte address behind the scenes without the Copy overhead.
- Use case: The default choice for passing objects/structures to avoid overhead.
- Pros/Cons: Fast and clean, but you must use `const` (e.g. `const T&`) if you want to prevent accidental modification.

#### pass by pointer
**Writing down the address of the box on a post-it note.**

- Mechanism: Passes the memory address of the variable as a value to the function.

- Memory View: A new 8-byte variable (on 64-bit systems) is created on the stack to store the address of the original object.

- Low-level Detail: - Indirection: The CPU must "dereference" the pointer to access the actual data, which can occasionally cause a cache miss.

- Nullability: Unlike references, pointers can be `nullptr`. This requires an explicit check (if (ptr)) to avoid segmentation faults, adding a branch to your code.

- Reassignable: You can change the pointer to point to a different memory address inside the function.

- Array Decay: When an array is passed, it loses its size information and "decays" into a raw pointer to the first element.

- Use Case: When the object is optional (can be null), when implementing C-style APIs, or when you need to change which object you are pointing to (re-seating).

- Pros/Cons: Highly flexible and efficient for large data, but risky due to null pointers and manual memory management requirements.

### Modern parameter design cheat sheet

### 1) Read-only input
- Small trivials: pass by value (`int`, `double`, raw pointers).
- Large objects: `const T&`.
- String-like view: `std::string_view` (if no ownership/copy needed).
- Array-like view: `std::span<const T>`.

```cpp
void print_name(std::string_view name);     // no copy, non-owning view
int sum(std::span<const int> nums);         // pointer+length done safely
```

### 2) Modifiable input/output
- Use `T&` when argument must be present and writable.
- Use pointer `T*` when argument is optional (`nullptr` allowed).

```cpp
void normalize(std::string& s);     // must exist
void maybe_fill(Buffer* out);       // optional output
```

### 3) Ownership transfer
- Prefer `std::unique_ptr<T>` by value to transfer exclusive ownership.
- Use `std::shared_ptr<T>` by value only when you need to share ownership.
- Use `const std::shared_ptr<T>&` only when you need to inspect the owner object itself.

```cpp
void set_logger(std::unique_ptr<Logger> lg);      // transfer ownership
void share_cache(std::shared_ptr<Cache> cache);   // share ownership
```

### 4) "Pass by value then move" pattern
For types frequently copied anyway, pass by value and move into member:

```cpp
class User {
public:
    explicit User(std::string name) : name_(std::move(name)) {}
private:
    std::string name_;
};
```

This keeps API simple and efficient for rvalue callers.

### Full source (09.pass_by_value_ref_ptr.cpp)

```cpp
#include <iostream>
#include <vector>

void work(std::vector<int>& data){
    for(int i = 0; i< data.size(); i++){
        data[i]++;
    }
    std::cout << &data << std::endl; // 0x16b21e6e0
}

int main(){
    std::vector<int> data = {1, 2, 3, 4, 5};
    std::cout << &data << std::endl; // 0x16b21e6e0
    work(data);
    std::cout << &data << std::endl; // 0x16b21e6e0
    for (int x : data){
        std::cout << x << " "; // 2 3 4 5 6
    }

    return 0;
}
```
