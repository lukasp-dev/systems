An object type `std::initializer_list<T>` is a lightweight proxy object that provides access to an array of objects of type `std::initializer_list` object.

```cpp
template< class T >
class initializer_list;
```

A `std::initializer_list` object is automatically constructed when:

1.  a brace-enclosed initializer list is used to list-initialize an object, where the corresponding constructor accepts an `std::initializer_list` parameter.
    - what happens when done `vector<int> v = {1, 2, 3};`?
        1. The compiler allocates a hidden, temporary backing array (typically in the read-only data segment(`.rodata`) or on the stack) containing the elements `[1, 2, 3]`.
        2. It creates a `std::initializer_list<int>` object. This is a lightweight **proxy** that stores a pointer to the start of the temporary array and its size (3).
        3. This proxy object is passed as an argument to the `std::vector` constructor, which then performs a deep copy of the elements into the vector's own heap-allocated memory.

    ```cpp
    #include <iostream>
    #include <initializer_list>

    class Basket {
        public:
            // this constructor is waiting for initializer_list
            Basket(std::initializer_list<const char*> items) {
                std::cout << "바구니에 담긴 개수: " << items.size() << std::endl;
                for (auto item : items){
                    std::cout << "- " << item << std:endl; 
                }
            }
    }

    int main() {
        // this {"apple", "grape", "orange"}
        // automatically becomes std::initializer_list<const char*> object.
        Basket myBasket = {"apple", "grape", "orange"};

        return 0;
    }
    ```
2. When a brace-enclosed list is used as a function argument or the right operand of an assignment.
    ```cpp
    void printNumbers(str::initializer_list<int> list) {/*...*/}
    // Automatically constructed for the function call
    printNumbers({10, 20, 30});
    ```
    or the right operand example...
    ```cpp
    std::vector<int> v;
    v = {4, 5, 6};
    ```
3. Binding to `auto` (including Range-for loops)
When you use the `auto` keyword with an assignment operator (`=`) followed by a braced-init-list `{...}`, the C++ compiler follows a special rule, it deduces the type of that variable as `std::initializer_list<T>`
    ```cpp
    // 'list' becomes std::initializer_list<double>
    auto list = {1.1, 2.2, 3.3};
    ```
    hidden auto in the range-for loop
    ```cpp
    for (int x : {1, 2, 3}){/*...*/}
    // Internally, {1, 2, 3,} is bound to a hidden 'auto' variable
    //which is a std::initializer_list<int>
    ```

### Question 1. Explain auto type deduction

`auto` type deduction is usually the same as template type deduction, but `auto` type deduction assumes that a braced initializer represents a `std::initializer_list`, and tempalate type deduction doesn't hold such premises.

### C++ 포인터 const와 auto 추론 정리

| 구분 | 코드 형태 (선언) | 의미 (해석) | 값 수정 (`*p = ...`) | 주소 수정 (`p = ...`) | `auto` 추론 결과 | 예시 설명 |
| :--- | :--- | :--- | :---: | :---: | :--- | :--- |
| **일반 포인터** | `int* p` | 데이터와 주소 모두 자유로움 | ✅ 가능 | ✅ 가능 | `int*` | 가장 기본적인 포인터 형태 |
| **상수를 가리키는 포인터** | `const int* p` | **가리키는 대상**이 상수 (Read-only) | ❌ 불가능 | ✅ 가능 | **`const int*`** | **[중요]** 데이터 보호를 위해 `const` 유지 |
| **상수 포인터** | `int* const p` | **포인터 변수 자체**가 상수 (고정 주소) | ✅ 가능 | ❌ 불가능 | `int*` | **[중요]** 복사본은 고정될 필요 없으므로 `const` 탈락 |
| **상수 데이터 상수 포인터**| `const int* const p` | **대상과 주소 모두** 상수 (완전 고정) | ❌ 불가능 | ❌ 불가능 | **`const int*`** | 데이터 보호(`const`)만 남고 주소 고정은 탈락 |

### 💻 코드 예시로 확인하기

```cpp
int x = 10;
int y = 20;

// 1. const int* (데이터가 상수)
const int* p1 = &x;
auto a1 = p1;        // a1 타입: const int*
// *a1 = 30;         // (Error) 데이터 수정 불가
a1 = &y;             // (Ok) 다른 주소 가리키기 가능

// 2. int* const (포인터가 상수)
int* const p2 = &x;
auto a2 = p2;        // a2 타입: int* (const가 사라짐!)
*a2 = 30;            // (Ok) 데이터 수정 가능
a2 = &y;             // (Ok) a2는 상수가 아니므로 주소 변경 가능

// 3. const int* const (둘 다 상수)
const int* const p3 = &x;
auto a3 = p3;        // a3 타입: const int*
// *a3 = 30;         // (Error) 데이터 수정 불가
a3 = &y;             // (Ok) 주소 변경 가능
