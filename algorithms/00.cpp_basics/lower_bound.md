## 🚀 Binary Search: Lower Bound 핵심 정리

### 1. r = m (범위 밖까지 보는 이유)

**"모든 요소가 조건을 만족하지 못할 경우"를 대비한 안전장치**

- **상황:** 포션이 5개 있을 때 (인덱스 0, 1, 2, 3, 4).
- **문제:** 만약 모든 포션이 실패라면? `r = 4`로 잡으면 루프는 0~4 사이에서만 답을 찾으려고 함.
- **해결:** `r = 5` (배열 크기 `m`)로 잡으면, 성공한 놈이 하나도 없을 때 `l`이 인덱스 5까지 밀려남.
- **결과:** `5(전체) - 5(찾은 위치) = 0`이 되어 예외 처리 없이 정확히 "0개 성공"이라는 정답 도출 가능.

---

### 2. r = mid (후보를 버리지 마!)

**"처음으로 성공한 놈(가장 왼쪽)"을 찾기 위한 전략**

- **상황:** 현재 `mid` 위치의 포션이 성공(`>= success`)함.
- **문제:** 이 `mid`가 1등(가장 왼쪽)일 수도 있는데, `r = mid - 1`을 하면 이 후보를 검색 범위에서 영구적으로 제외하게 됨.
- **해결:** `r = mid`로 설정하여 **"지금 너 성공했으니까, 일단 너를 경계선으로 잡고 더 왼쪽도 살펴볼게"**라고 후보를 유지함.

---

### 💡 표준 구현 코드 (C++)

```cpp
int l = 0, r = m; // m은 potions.size()
while (l < r) {
    int mid = l + (r - l) / 2;

    // 1. 곱셈 시 오버플로우 방지를 위해 (long long) 캐스팅 필수
    // 2. 현재 mid가 성공했다면 'r = mid'로 후보 유지
    if ((long long)potions[mid] * spell >= success) {
        r = mid;
    } else {
        l = mid + 1; // 실패했다면 무조건 오른쪽 확인
    }
}
// 결과: l은 '처음으로 성공한 인덱스'를 가리킴
return m - l;
```
